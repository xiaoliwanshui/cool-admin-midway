## 项目简介

Cool Admin Midway 是一个基于 **Node.js** 和 **TypeScript** 的现代化企业级后台权限管理系统，旨在通过现代化技术栈和 AI 编码能力，快速构建和迭代后台管理系统。该项目以 Midway 框架为核心，提供了一套完整的解决方案，支持 AI 编码、流程编排、模块化、插件化以及极速开发 CRUD 功能。

项目的核心定位是为开发者提供一个高效、灵活且可扩展的后端管理平台，能够快速搭建各类管理后台，满足企业级应用的需求。通过集成 AI 技术，开发者可以实现从 API 接口到前端页面的一键生成，显著提升开发效率。同时，系统支持多租户、权限控制、任务调度等企业级功能，适用于中后台系统开发、微服务架构集成和快速原型构建等多种场景。

## 核心架构与技术栈

### 技术选型

#### 后端技术栈
- **运行时**: Node.js (>=18.x)
- **编程语言**: TypeScript (5.8.2)
- **Web 框架**: MidwayJS + Koa.js
- **ORM 框架**: TypeORM (定制版 @cool-midway/typeorm@0.3.20)
- **缓存**: Redis + cache-manager-ioredis-yet
- **任务调度**: Cron
- **文件上传**: @midwayjs/upload
- **参数验证**: @midwayjs/validate
- **API 文档**: Swagger
- **安全认证**: JWT + SVG 验证码

#### 前端技术栈
- **框架**: Vue.js 3.x
- **UI 组件库**: Element-Plus
- **状态管理**: Pinia
- **路由管理**: Vue-Router
- **渲染方式**: JSX

#### 数据库支持
- **MySQL** (>=5.7，推荐 8.0)
- **PostgreSQL**
- **SQLite**

### 架构设计

#### 设计模式
- **模块化设计**: 代码按功能模块划分，便于维护
- **插件化设计**: 通过插件扩展功能（支付、短信、邮件等）
- **装饰器模式**: 使用 TypeORM 和 Midway 框架的装饰器特性
- **依赖注入**: Midway 框架内置依赖注入机制

#### 性能优化
- **Redis 缓存**: 数据缓存和会话管理
- **数据库优化**: TypeORM 查询优化
- **多租户查询**: 动态注入查询条件

#### 安全特性
- **JWT 认证**: 用户身份认证和授权
- **权限控制**: 基于角色的权限管理
- **日志记录**: 完整的操作日志和审计跟踪

## 模块化架构设计

Cool Admin Midway 采用模块化设计，将代码按功能模块进行划分，每个模块都有独立的配置和依赖，支持热插拔和动态加载。这种设计使得系统更加清晰、易于维护，并且可以根据需求灵活地添加或移除模块。

### 主要模块
- **application**: 应用管理模块
- **base**: 基础模块（权限、日志、登录等）
- **demo**: 示例模块（商品管理、队列等）
- **dict**: 数据字典模块
- **echart**: 图表统计模块
- **plugin**: 插件管理模块
- **recycle**: 回收站模块
- **space**: 存储空间模块
- **swagger**: API 文档模块
- **task**: 任务调度模块
- **user**: 用户管理模块
- **video**: 视频管理模块

每个模块都包含控制器（controller）、实体（entity）、服务（service）等组件，遵循 MVC 模式，确保代码结构清晰、职责分明。

## 插件化扩展机制

Cool Admin Midway 支持插件化架构，允许开发者通过插件扩展系统的功能。插件化设计提供了丰富的插件（如支付、短信、邮件等），并且支持标准化的插件开发接口，便于开发者创建和管理自己的插件。

### 插件管理
- **丰富插件**: 提供支付、短信、邮件等常用功能插件
- **易于扩展**: 标准化的插件开发接口，简化插件开发过程
- **一键安装**: 支持插件的在线安装和管理

插件通过 `@CoolController` 和 `@CoolTag` 等装饰器进行注册和配置，确保插件能够无缝集成到系统中。此外，插件还可以通过 `pluginService` 进行实例化和调用，实现灵活的功能扩展。

## 企业级功能支持

### 多租户支持
Cool Admin Midway 采用全局动态注入查询条件的方式，实现真正的多租户隔离。通过 `TenantSubscriber` 订阅者，系统能够在查询时自动注入租户条件，确保数据安全。

- **数据隔离**: 自动注入租户条件，确保数据安全
- **配置灵活**: 支持多种租户模式和权限策略
- **性能优化**: 动态查询优化，不影响系统性能

### 多语言支持
基于大模型自动翻译，无需更改原有代码。系统利用 AI 大模型进行高质量翻译，支持语言包的动态加载和更新，确保应用在全球范围内的一致性和可用性。

- **智能翻译**: 利用 AI 大模型进行高质量翻译
- **无侵入性**: 不需要修改业务代码逻辑
- **实时更新**: 支持语言包的动态加载和更新

### 原生打包
支持打包成 exe 等安装包，可直接运行在 Windows、Mac、Linux 等操作系统上。通过原生打包，企业可以快速部署内部系统，提高部署效率和安全性。

- **跨平台**: 支持 Windows、macOS、Linux 多平台
- **企业部署**: 适合企业内部系统的快速部署
- **安全可靠**: 独立可执行文件，不依赖运行时环境

## 可扩展性示例

### base 模块
`base` 模块是系统的基础模块，包含了权限、日志、登录等核心功能。通过 `BaseSysUserController` 和 `BaseSysUserService`，系统实现了用户管理、权限控制和日志记录等功能。

```typescript
@CoolController({
  api: ['add', 'delete', 'update', 'info', 'list', 'page'],
  entity: BaseSysUserEntity,
  service: BaseSysUserService,
  insertParam: ctx => {
    return {
      userId: ctx.admin.userId,
    };
  },
})
export class BaseSysUserController extends BaseController {
  @Inject()
  baseSysUserService: BaseSysUserService;

  @Post('/move', { summary: '移动部门' })
  async move(
    @Body('departmentId') departmentId: number,
    @Body('userIds') userIds: []
  ) {
    await this.baseSysUserService.move(departmentId, userIds);
    return this.ok();
  }
}
```

### user 模块

```typescript
export class AdminUserInfoController extends BaseController {}
```

### video 模块
`video` 模块负责视频管理，包括视频信息、播放线路、专辑管理等。通过 `AdminVideoController` 和 `VideoEntity`，系统实现了视频信息的增删改查功能。

```typescript
@CoolController({
  api: ['add', 'delete', 'info', 'list', 'page', 'update'],
  entity: VideoEntity,
  insertParam: ctx => {
    return {
      createUserId: ctx.admin.userId
    };
  },
  pageQueryOp: {
    keyWordLikeFields: ['title'],
    fieldEq: [
      'category_id',
      'year',
      'language',
      'region',
      'play_url_put_in',
      'category_pid',
      'searchRecommendType'
    ],
    addOrderBy: {
      updateTime: 'desc'
    }
  }
})
export class AdminVideoController extends BaseController {
  @Inject()
  videosService: VideosService;

  @CoolTag(TagTypes.IGNORE_TOKEN)
  @Post('/sort', { summary: '排序' })
  async sort(@Body() body): Promise<unknown> {
    try {
      return this.ok(await this.videosService.sort(body));
    } catch (error) {
      return this.fail(error);
    }
  }
}
```
